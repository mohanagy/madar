## How Dub resolves a short-link click

The whole flow runs in **Next.js middleware** (Node runtime, runs before any page render) and follows three stages: request handling â†’ analytics tracking â†’ destination redirect.

### 1. Request handling & routing

**`apps/web/middleware.ts`** is the entry point. For every incoming request it calls **`parse()`** (`lib/middleware/utils/parse.ts`) to extract the `domain` (from the `host` header), `key`, and `fullKey` from the URL path. It then routes by hostname: app/API/admin/partners hostnames go to their own middlewares, `/stats/` and `/.well-known/` paths rewrite to internal pages, and anything that isn't a valid URL (i.e., a normal short link) falls through to **`LinkMiddleware(req, ev)`** (`middleware.ts:89`).

### 2. Link lookup & validation â€” `lib/middleware/link.ts`

`LinkMiddleware` does the resolution work:

- **Normalizes the key**: puny-encodes it, lowercases it unless the domain is case-sensitive (`isCaseSensitiveDomain`), handles the `+` "inspect mode" suffix, and maps an empty key to `_root` (`link.ts:47-64`).
- **Looks up the link**: first hits the Redis cache via **`linkCache.get({ domain, key })`** (`lib/api/links/cache`). On a miss it falls back to PlanetScale with **`getLinkViaEdge()`** (`lib/planetscale`), then back-fills the cache (`linkCache.set`), enriching partner links with `getPartnerEnrollmentInfo` (`link.ts:86-133`). A genuine miss rewrites to `/{domain}/notfound`.
- **Applies link rules in order** (`link.ts:135-253`): unsupported `.php` keys, password protection (rewrite to `/password/[linkId]` until the `pw` cookie/param matches â€” ensuring no click is tracked on protected links), banned workspace (`LEGAL_WORKSPACE_ID`), disabled links, and expiration (redirect to `expiredUrl` or rewrite to `/expired`).
- **Resolves the destination URL**: runs A/B test selection via **`resolveABTestURL()`**, picking `testUrl` or the link's default `url` (`link.ts:154-159`).
- **Establishes the click identity**: reads or creates the `dub_id_{domain}_{key}` cookie. If absent and the link needs conversion tracking (`trackConversion`, partner link, or Singular/AppsFlyer URL â†’ `shouldCacheClickId`), it checks `recordClickCache` keyed by an identity hash; otherwise it generates a fresh `nanoid(16)` `clickId` (`link.ts:255-280`).

### 3. Analytics tracking â€” `recordClick()` in `lib/tinybird/record-click.ts`

In each redirect branch, the middleware fires **`recordClick({...})`** inside `ev.waitUntil(...)` so tracking happens asynchronously without delaying the redirect. `recordClick`:

- Bails on `dub-no-track` header/param, bot detection (`detectBot`, skipped for deep links), and **deduplicates** one click per IP+domain+key per hour via `recordClickCache` (unless `skipRatelimit`) (`record-click.ts:61-108`).
- Detects QR scans (`detectQr` â†’ `trigger = "qr"`), and assembles `clickData` from geo headers (`geolocation`, `x-vercel-ip-*`), IP (omitted for EU countries), and parsed user-agent (device/browser/OS/engine), plus referer (`record-click.ts:110-169`).
- In a `Promise.allSettled` it: **POSTs the event to Tinybird** (`/v0/events?name=dub_click_events`), sets the dedup cache, runs `UPDATE Link SET clicks = clicks + 1, lastClicked = NOW()` on PlanetScale, publishes workspace-usage and partner-activity events (with direct-DB fallbacks), and optionally fires `link.clicked` **webhooks** (`record-click.ts:177-289`). If `shouldCacheClickId`, the click data is also stashed in Redis for 5 min so later `/track/lead` conversion calls can find it.

### 4. Destination redirect

After (well, alongside) `recordClick`, the middleware builds the final URL with **`getFinalUrl(url, { req, clickId, via })`** (`lib/middleware/utils/get-final-url.ts`), which merges incoming query params onto the destination, supports a `redir_url` override, appends the `dub_id` click ID (or Stripe `client_reference_id`), adds `via` for partner attribution, and handles AppsFlyer/Singular/Google-Play tracking-param injection.

The branch chosen (`link.ts:313-614`) depends on the link's config:
- **Bot + proxy** â†’ rewrite to `/{domain}/{key}/proxy` (OG-tag page).
- **Custom URI scheme** â†’ rewrite to `/custom-uri-scheme/...`.
- **Cloaking (`rewrite`)** â†’ rewrite to `/cloaked/...` (URL stays masked).
- **iOS / Android targeting** â†’ redirect to the platform URL (with an interstitial `/deeplink/...` splash page for App Store / Play Store links).
- **Geo targeting** â†’ redirect to `geo[country]`.
- **Default** â†’ `NextResponse.redirect(finalUrl)` with status `301` for `_root`, else `302`.

Every branch wraps its response in **`createResponseWithCookies(response, cookieData)`** so the `dub_id` cookie (and A/B test variant) is persisted on the redirect for downstream conversion attribution.
