## What the graph evidence supports

**Notification delivery (well-covered, `pack_confidence: direct`)**
- Outbound document mail goes through a pluggable Nodemailer-style `Transport`. The Cloudflare path is `MailChannelsTransport` (`packages/email/transports/mailchannels.ts:28`).
- Its `.send()` (`:47`) validates required `to`/`from`, normalizes every address field (`to`/`cc`/`bcc`/`from`) via `.toMailChannelsAddresses()` (`:125`), then `POST`s the payload to the MailChannels API (`https://api.mailchannels.net/tx/v1/send`), attaching `X-Auth-Token` when an API key is configured.

**Sender identity / email config (medium confidence)**
- Which verified domain/address a document is sent *from* is resolved through tRPC routers: `findEmailDomains()` (admin, `packages/trpc/server/admin-router/find-email-domains.ts:24`) and the enterprise equivalents `findOrganisationEmailDomains()`, `findOrganisationEmails()`, `getOrganisationEmailDomain()` — these gate sending on an organisation's verified `EmailDomain`/`OrganisationEmail` records.
- A `webhook-router/router.ts` exists, indicating document-lifecycle events also fan out to webhook subscribers alongside email.
