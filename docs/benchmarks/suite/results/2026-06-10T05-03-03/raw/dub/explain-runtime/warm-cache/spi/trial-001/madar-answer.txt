## How Dub resolves a short-link click

**1. Request handling** — An incoming request hits `middleware.ts:35` (`middleware()`), which dispatches link requests to `LinkMiddleware()` in `lib/middleware/link.ts:40`. The middleware resolves the short key to its link record and, for deep links, captures click context via `cacheDeepLinkClickData()` (`lib/middleware/utils/cache-deeplink-click-data.ts:10`).

**2. Analytics tracking** — Before redirecting, the click is recorded to Tinybird via `recordClick()` in `lib/tinybird/record-click.ts:28`, which logs the click event for analytics.

**3. Destination redirect** — The visitor is sent to the link's target via `NextResponse.redirect` (`lib/middleware/link.ts:77`, with the middleware-level redirect at `middleware.ts:74`).

**Key files**
- `middleware.ts` — entry dispatch
- `lib/middleware/link.ts` — link resolution + redirect (`LinkMiddleware`, `NextResponse.redirect`)
- `lib/middleware/utils/cache-deeplink-click-data.ts` — click-context capture
- `lib/tinybird/record-click.ts` — analytics tracking (`recordClick`)

Flow: `middleware()` → `LinkMiddleware()` → `cacheDeepLinkClickData()` → `recordClick()` → `NextResponse.redirect` to destination.
