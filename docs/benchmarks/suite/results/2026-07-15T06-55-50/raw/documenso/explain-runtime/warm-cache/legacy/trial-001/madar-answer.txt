## Documented flow

**1. Send preparation — document data**

**2. Recipient creation**
`createEnvelopeRecipients()` (`server-only/recipient/create-envelope-recipients.ts:30`):
- Resolves and authorizes the envelope via `getEnvelopeWhereInput` (rejects if not found or already `completedAt`).
- Guards action-auth: recipients with `actionAuth` require the team's `cfr21` org flag.
- Lowercases emails, then inside a **`prisma.$transaction`** creates each recipient with `createRecipientAuthOptions()` (access/action auth), a unique `token: nanoid()` (the signing-link token), and, for `EnvelopeType.DOCUMENT`, writes a `RECIPIENT_CREATED` audit log via `createDocumentAuditLogData()`.
- Returns recipients mapped through `mapRecipientToLegacyRecipient()`.

**3. Initial signing state (set at creation)** — `create-envelope-recipients.ts:104-105`:
- CC recipients start `sendStatus = SENT`, `signingStatus = SIGNED` (they never need to act).
- All other roles start `sendStatus = NOT_SENT`, `signingStatus = NOT_SIGNED`, awaiting dispatch/signing.
