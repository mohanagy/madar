## Verified flow

**Send preparation → envelope items**
`UNSAFE_createEnvelopeItems` (`server-only/envelope-item/create-envelope-items.ts`) prepares each uploaded file for sending: it inserts saved form values (`insertFormValuesInPdf`), normalizes and flattens the PDF (`normalizePdf`, flattened unless it's a `TEMPLATE`), strips/extracts placeholders (`extractPdfPlaceholders`), and uploads the cleaned PDF (`putPdfFileServerSide`). Inside one `prisma.$transaction` it creates the `EnvelopeItem` rows, writes `ENVELOPE_ITEM_CREATED` audit logs, and — if recipients already exist — converts placeholders into `Field` rows, matching each placeholder to a recipient by signing order (`convertPlaceholdersToFieldInputs` + `findRecipientByPlaceholder`).

**Recipient creation + initial signing state**
`createEnvelopeRecipients` (`server-only/recipient/create-envelope-recipients.ts`) loads the envelope, blocks if it's already `completedAt`, and enforces the `cfr21` org flag before allowing action-auth. In a transaction it creates each `Recipient` with a `nanoid()` signing token and auth options from `createRecipientAuthOptions`. The key signing-state rule: `CC` recipients start `SendStatus.SENT` / `SigningStatus.SIGNED` (nothing to sign), while all other roles start `NOT_SENT` / `NOT_SIGNED`. For `DOCUMENT`-type envelopes it writes a `RECIPIENT_CREATED` audit log, then returns recipients mapped to the legacy shape.
